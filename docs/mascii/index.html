The mascii translator

This page documents my years of failure trying to write interpreters to ease my text writing such as LaTeX. I'll begin with introducing you to mascii. Then I'll show you elproc and finally I'll point to softwares that can do the job properly. Come with me on this adventure. The word ``mascii'' stands for ``mathematical ascii.'' That's a few set of rules that I use when I write mathematics that I intend to convert to TeX or LaTeX.

The program mascii does the translation. For example, the program mascii can read ``let f(x) be a function from R -> R such that f(x + y) = f(x) <*> f(y)'' and translate that to ``let $f(x)$ be a function from $\mathbb{R} \to \mathbb{R}$ such that $f(x + y) = f(x) \cdot f(y)$.''

This page does not describe the language that mascii understands; this page just announces the program. To learn about mascii translation capabilities, you must read the documentation of the software included in the package of the software.

Because mascii makes a lot of mistakes, it is interesting to use it slowly on small regions, as you write, so that you can learn about its mistakes --- and learn to cope with them. For example, currently, if you write two fractions in the same line, mascii will do the wrong thing, so it's convenient to avoid writing two fractions on the same line. But, before you know what to do, you must know what is a mascii fraction. A mascii fraction is any sequence of bytes that matches the regular expression

({[^}].*}[ ]*/[ ]*{[^}].*})

which 1/2 does not match. Not matching 1/2 is interesting because quite often I want to write something like ``(x + 1)^{1/2}'' and I don't want mascii to tell TeX that the ``1/2'' is supposed to be a fraction in the shape ``numerator on top, a dash underneath, and denominator on the bottom.'' Examples of fractions that get this shape are

{ numerator } / { denominator }.

Spaces in between things are allowed.

Interaction with the GNU EMACS

I like to use mascii from inside my GNU EMACS. As I write a paragraph, or a few paragraphs, I select a region that encloses the text I wrote, then I press the key a combination of keys which causes the GNU EMACS to feed that text to mascii and the GNU EMACS replaces the region with the output of mascii.

Notice the pictures. On the first, I have written a piece of text; on the second, I've selected a region under the GNU EMACS. On the third, I pressed a special key combination and the GNU EMACS has replaced the region with the output from mascii.
